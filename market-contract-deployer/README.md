# market-contract-deployer

One-shot container that replays the committed Alkahest deployment
transactions onto a target Anvil-compatible RPC and exits.

After Phase 4 of the pluggable-identity refactor this is the only
contract suite the deployer ships — ERC-8004 contracts were removed
along with the rest of the ERC-8004 code paths.

## What it does

1. **Alkahest replay** — runs `deploy_alkahest.py`, which reads
   `alkahest-transactions.json` and replays every transaction via
   `eth_sendTransaction`, deploying the Alkahest infrastructure
   (ERC-20 mock + escrow + arbiter + EAS contracts).

## Usage

### As part of docker compose

`contracts-deploy` is a one-shot init container in `docker-compose.yml`.
It runs, exits, and downstream services start once it completes.

```bash
docker compose up
```

### Standalone

```bash
docker build -t arkhai:contract-deployer .
docker run --network=host \
  -e RPC_URL=http://localhost:8545 \
  arkhai:contract-deployer
```

## Files

- `deploy_alkahest.py` — stdlib-only Python script that replays
  transactions against the configured RPC.
- `alkahest-transactions.json` — committed artifact. Regenerated by
  `storefront/scripts/generate_alkahest_transactions.py` whenever the
  alkahest-py wheel version changes.
- `deploy-local.sh` — container entrypoint; just calls `deploy_alkahest.py`.
